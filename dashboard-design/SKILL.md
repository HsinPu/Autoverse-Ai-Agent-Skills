# Dashboard Design

Use this skill to design dashboards that answer real questions.

## Workflow

1. Identify the primary questions the dashboard must answer.
2. Group metrics into cards, sections, or lanes by importance and workflow.
3. Decide which elements need prominence and which can stay compact.
4. Use spacing, grid, and contrast to make scanning fast.
5. Verify the layout works on both large and smaller screens.

## Rules

- Do not make the dashboard decorative first and useful second.
- Keep the number of visual priorities low.
- Use cards only when they improve scanning or grouping.
- Prefer decision-making clarity over filler density.

## Handoff

- For implementation, use `frontend-design` or `responsive-design`.
- For dashboard component styling, use `ui-styling`.
