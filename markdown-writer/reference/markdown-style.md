## 語言與用語

- **繁體中文**：正文、標題、列表與說明一律使用繁體中文。
- **中英文混和**：技術術語、專有名詞、指令與程式名稱保留英文；敘述以中文為主，首次出現的術語可加括號補英文，例如「請求（request）」「端點（endpoint）」。
- 程式碼、指令、檔名、變數名等維持英文或專案既有命名，不翻譯。

---

## 標題

- 使用 `#`～`######`，一級一個 `#`，對應 HTML `<h1>`～`<h6>`。
- 標題前後各留一空行。
- 階層依序、不跳級：`#` → `##` → `###`。
- 標題後可選加 `{#anchor}` 以自訂錨點（視渲染器支援）。

```markdown
# 一級標題
## 二級標題
### 三級標題
```

---

## 列表

- **無序**：同一文件內統一用 `-`（或 `*`），縮排兩空格表示巢狀。
- **有序**：用 `1.` 開頭即可，渲染器會自動編號；需延續編號時可寫 `2.`、`3.`。
- 列表項與前段、列表與列表之間留空行可提升可讀性。

```markdown
- 項目一
- 項目二
  - 子項目
1. 第一步
2. 第二步
```

## 程式碼

- **行內**：反引號 `` `code` ``，用於指令、變數、檔名、關鍵字。
- **區塊**：fenced code block，標註語言以啟用語法高亮。

````markdown
```bash
npm install
```

```python
def hello():
    print("Hello")
```
````

- 若內容含 `` ``` ``，外層可用更多反引號包住（如 ` ``` ` 四個反引號）。

## 連結與圖片

- 連結：`[顯示文字](URL "選填 title")`
- 圖片：`![替代文字](URL "選填 title")`
- 參考式：`[文字][ref]`，文末定義 `[ref]: https://...`
- 錨點：多數渲染器支援 `#標題-slug`，標題中的空格、英文大小寫會轉成 `-` 或小寫。

---

## 表格

- 表頭列、分隔列（`|---|---|`）、資料列；分隔列至少各欄一個 `-`。
- 對齊：`:---` 左、`:---:` 中、`---:` 右。
- 欄位內若有 `|`，用 HTML `<code>` 或避免使用 `|`。

```markdown
| 欄 A | 欄 B |
|------|------|
| a1   | b1   |
| a2   | b2   |
```

---

## 強調與行內元素

- **粗體**：`**文字**` 或 `__文字__`
- *斜體*：`*文字*` 或 `_文字_`
- 刪除線（GFM）：`~~文字~~`
- 同一文件內選一種風格（例如粗體只用 `**`）以維持一致。

---

## GFM 與相容性

- **GitHub Flavored Markdown (GFM)**：表格、刪除線、自動連結、fenced code 語言標籤、任務列表 `- [ ]` / `- [x]`。
- 若產出需在 GitHub、GitLab、常見靜態站使用，以 GFM 為準即可。
- 避免依賴少數平台專屬擴充（如特定 wiki 語法），以利移植。

## 檢查清單（輸出前）

- [ ] 全文使用繁體中文，採中英文混和（術語保留英文）
- [ ] 標題階層連續、不跳級
- [ ] 列表符號與縮排一致
- [ ] 程式碼區塊有語言標籤（若需高亮）
- [ ] 連結可點、圖片有 alt
- [ ] 表格對齊、欄位內無破壞語法的 `|`
- [ ] 術語與格式（粗體/程式碼）用法一致
