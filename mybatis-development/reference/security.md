# 安全（Security）

## `#{}` vs `${}`

- `#{...}`：會產生 PreparedStatement 參數（`?`），是預設且幾乎總是正確的選擇
- `${...}`：字串直接注入 SQL（不會 escape）；官方文件明確警告：若接收使用者輸入會造成 SQL Injection

## 動態識別字的安全做法

- 動態欄位名/表名/ORDER BY：
  - 只允許白名單值（例如 enum/固定集合）
  - 不接受任意字串；不要把使用者輸入直接丟進 `${}`
  - 若需要動態排序：用白名單 map（API 參數 -> 真正欄位名），再帶入 `${}`

## Dynamic SQL 與輸入

- 動態條件用 `<if>` + `#{}`，不要改用 `${}` 來省事
- `IN` 條件用 `<foreach>` + `#{item}`；不要把整段 `1,2,3` 字串當成 `${ids}`

## 交易與權限（資料庫層）

- MyBatis 幫你處理參數化，不代表可以忽略 DB 權限：應用帳號採最小權限、避免用 DBA 權限跑應用
